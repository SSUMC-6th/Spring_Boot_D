- Web Server
정적인 데이터를 처리하는 서버. 이번에 실습한 nginx가 그중 일부!
- Web Application Server
동적인 데이터를 처리하는 서버. 스프링부트에서의 Tomcat WAS에 해당된다. 주로 DB와 연동할때 자주 쓰인다.
- Reverse Proxy
같은 컴퓨터의 다른 서버로 요청을 보내주는 대리자.

? 그렇다면 모든 서버를 WAS로 만들면 되는거 아닌가??
- WAS는 DB 조회 및 다양한 로직을 처리하는 데 집중해야 함. 단순한 정적 콘텐츠는 웹 서버에게 맡기며 기능을 분리해 서버 부하를 방지!
따라서 WAS는 서버 사이드에 관해서 로직을 처리하는데에만 부하를 받고, 역할 분담이 잘 되는 장점.
