<AWS>
Amazon Web Service의 줄임말

<리전과 가용영역>
세계적으로 서버를 구축하는 PC들을 곳곳에 나눠서 둔것. 가용영역은 리전을 한번더 분산해서 배치. 이유는 너무 멀리떨어져 있어서, 통신이 느려지거나, 자연재해등 PC에 손상을 입었을경우를 대비할 수 있기때문. DB에서 RAID기법을 생각하면 쉽다!

<서브네팅>
일반적을 A, B, C 클래스를 할당하게 되는데, 그냥 쓰게 되면 낭비되는 주소가 너무 많다! 그래서 서브넷 마스크를 이용해서 네트워크ID와 호스트ID를 분리한다.
부족한 IP주소를 더욱 알차게 활용하기 위해서, 네트워크 ID개수를 늘려, 호스트  ID를 잘 활용할수 있게해줌.

<라우팅>
통신하면서 패킷을 출발지, 목적지와 같은 여러가지 정보를 종합해서, 최적의 통신경로를 찾는 과정. 라우팅 테이블을 이용한다.
평소에 우리가 목적지에 가기 위해서 지도앱에서 목적지를 입력하고 경로를 파악해서 최대한 빨리 갈 수있는 경로를 찾는 것을 생각하면 간단하다.
패킷이란 데이터를 전송하는 하나의 단위이다.
정적 라우팅(수동)과 동적 라우팅(자동)으로 분리해서 볼 수있다. 

<VPC>
virtual private cloud의 줄임말로, 서브네팅을 활용하여, ip주소를 할당받을 수 있다. 기존 네트워크와 매우 유사한 가상의 네트워크이다.

<사설 IP주소>
어떤 네트워크안에서만 사용되는 고유한 주소. 로컬 IP라고도 불림. 보안성이 높고, 부족한 ip주소를 충당 해준다.
사실 IPv6주소 체계를 사용한다면 사용하지 않아도 된다. 그러면 대체 왜 쓰는가? 당연히 IPv6주소 체계는 존재하지만, 이때까지 쭉 IPv4 주소체계를 사용해 왔기 때문에 IPv6주소 체계로 바꾸는 것은 정말 많은 비용이 드는 한계가 있기 때문이다. 어떻게든 IPv4를 쓰려고 하는 전략으로 볼 수있다.

<포트포워딩>
공용 ip를 부여받은 하나의 기기에서 사설ip를 부여받은 기기들에게 데이터를 전달해주는것.

<NAT프로토콜>
Network Address Translation의 줄임말이다.
공용 ip의 개수가 부족하기 때문에, 하나의 공용 ip에서 연결된 각 기기들에게 port번호부여, 사설ip를 만들어준다.

<NAT 와 포트포워딩의 차이점?>
내부(사설) 주소를 외부(공인) 주소로 바꾸어준다는 개념 또는 기능들이라는 공통점을 가지지만 엄연히 다른 개념이다.
NAT는 여러 사설 IP 주소를 하나의 공인 IP 주소로 Mapping 함으로써 각 호스트마다 공인 IP 주소를 할당할 필요가 없다. 따라서 공인 IP 주소를 절약할 수 있다.
포트 포워딩은 NAT에 포트 정보를 추가한 개념으로, 각각의 내부 호스트에 독립적인 포트 번호를 부여하여 하나의 외부 IP 주소로 전송해도 포트 번호로 호스트를 구별할 수 있게 도와준다. 이 개념에서 포트를 고정하여 외부에서 내부로 접속이 가능하게끔 도와주는 개념이다.

<포트번호>
NAT 프로토콜에서 설명한 내용이다. 사설 ip주소를 가리킨다. 하나의 공유기의 연결된 기기들을 구분해주는 역할을 한다.
