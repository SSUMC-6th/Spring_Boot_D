- API
Application Programming Interface의 약자이다.
어렵게 생각할 것없이 Interface가 무엇인가에 대해서 생각해보면 예를 들어 우리가 형광등을 켠다고 생각해보자. 우리는 불을 켜는 스위치를 누르기만하면된다. 이게 바로 인터페이스이다. 그리고 이걸 내가 구현한 비즈니스 로직에 적용시키면 그게 바로 API이다!

- Rest API
HTTP를 기반으로 한 웹 서비스를 포괄적으로 의미한다.

- API Endpoint
어떤 HTTP메소드를 사용했는가? + URL을 의미한다.

- URL
Uniform Resource Locator의 약자로 네트워크상에서의 위치를 나타낸다. URI (Identifier)에 포함되는 관계이다.

- Path Variable
여러개의 대상이 있을때, 지정하고싶은 대상을 식별할수 있는 값이다.
{}(중괄호)를 써서 표현하고, 보통 기본 키로 활용해서 객체를 구분짓는다.

- Query String
윈도우 상에서 ctrl + f 를 통해 검색을 하는것을 생각하면 이해하기 좋다. 따라서 HTTP GET 메서드에 활용된다.
Endpoint 이후에 "?" 기호로 구분을해서 사용하며, "&" 기호를 구분자로 사용하여 여러개의 쿼리스트링을 질의 할수도있다.

- Request Body
주로 보안상의 이유 혹은 전달해야할 데이터의 양이 많고, 정형화 되어야하는 요청에서 쓰인다.
가장 많이 사용하는 Json 형식이 있고, form-data, raw등 많은 형식으로 원하는 요청을 받을때 쓰인다.

- Request Header
메타데이터를 담는 부분이다.
메타데이터란? 데이터의 데이터 즉, 전체적인 설명부분이다. 예시로 책을 든다면, 책의 페이지 수, 책의 가격, 책의 저자 등이 Request Header에 대응한다고 볼수있다.
주로 커뮤니티 서비스에서의 JWT로 사용자 인증토큰을 식별할때 이 정보를 담는다.
